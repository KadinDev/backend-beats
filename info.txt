DEPLOY VERCEL

Este projeto foi preparado para Vercel com o minimo de infraestrutura:
HTML/CSS/JS estatico no front, Serverless Functions pequenas para admin/API, e Vercel Blob para os MP3.

Agora tambem existe um painel em /admin.html para enviar e deletar musicas novas.
Uploads novos usam Vercel Blob via client upload, entao o arquivo vai direto do navegador para o Blob.
Isso evita trafegar MP3 pela funcao Node.

Endpoint usado pelo app:

  /api/tracks

Painel:

  /admin.html

Variaveis de ambiente na Vercel:

  ADMIN_PASSWORD=uma-senha-forte
  BLOB_READ_WRITE_TOKEN=token-do-vercel-blob

Cadastro manual do admin:

  1. Abra o projeto na Vercel.
  2. Va em Settings > Environment Variables.
  3. Crie ADMIN_PASSWORD com uma senha forte.
  4. Crie/conecte um Blob Store e garanta BLOB_READ_WRITE_TOKEN.
  5. Faca um novo deploy.

Com esse modelo existe um unico admin: quem souber a senha ADMIN_PASSWORD.

Comandos:

  npm install
  npx vercel
  npx vercel --prod

Configuracao importante:

- Todos os audios ficam no Vercel Blob, nao no Git.
- O upload vai direto do navegador para o Blob; a funcao so gera permissao curta.
- A API /api/tracks retorna titulo, url, pathname, tamanho e data.
- O app mobile deve cachear os audios no dispositivo para cada usuario baixar cada beat o minimo possivel.
- Nao use funcao Node para streaming de MP3 se a prioridade for economia.

Atencao:

O conteudo total dos MP3 tem cerca de 230 MB. Na Vercel, o custo principal passa a ser bandwidth/CDN.
Se o app crescer muito, o proximo passo para economizar e controlar custo e mover os MP3 para um storage/CDN barato
como Cloudflare R2 + cache, mantendo estes mesmos caminhos ou redirecionamentos.
